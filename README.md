# A股每日扫描 · 手把手部署教程（零基础版）

全程只需要点鼠标和复制粘贴，不需要命令行。总耗时约 15 分钟（不含首次数据下载的等待）。

## v2 更新（2026-07-06）

- **日/夜主题**：右上角按钮切换，浏览器记住你的选择
- **今日候选页交互筛选**：股价区间 + 行业板块（只影响显示，不改选股规则）
- **划掉+标原因**：触发「MACD红柱首日」的股票全部列出，被规则刷掉的划横线折叠在下方并标注全部原因
- **新硬规则**（已写进回测保持一致）：换手率<2%划掉、>25%黄色警告；市值<50亿划掉；基本面任一恶化信号或综合分<40划掉
- **已部署过 v1 怎么升级**：把新 zip 里的文件重新走一遍第三、四步上传（同名文件自动覆盖），已下载的历史数据不受影响、无需重拉

## 规则档案（与用户逐条确认于 2026-07）

| 类别 | 规则 |
|---|---|
| 入场信号 | MACD 柱由绿转红首日（国内口径，柱=2×(DIF−DEA)） |
| 趋势/量能 | 价>20日线；量>5日均量；成交额>3亿 |
| 强弱 | 当日涨幅强于沪深300 |
| 流动性/规模 | 换手率≥2%（>25%仅警告）；市值≥50亿 |
| 排除 | ST；上市不足60个交易日 |
| 基本面划掉 | 净利润同比降超20% / 现金流质量差 / 应收增速远超营收 / 毛利率恶化 / 综合分<40 |
| 卖出（回测） | -5%止损 或 MACD转绿次日开盘卖；T+1约束 |

网站包含三个页面：**今日候选**（按你确认的 MACD 红柱首日 + 六项过滤规则每日自动筛选）、**基本面评分**（0-100 分 + 情绪杀/基本面杀归因标签）、**策略回测**（纯 MACD vs 加过滤两套规则的历史对比）。

## 第一步：注册 GitHub（已有账号可跳过）

1. 打开 https://github.com/signup
2. 按提示填邮箱、密码、用户名，完成验证。**记住你的用户名**，最终网址里会用到。

## 第二步：创建仓库

1. 登录后点右上角的 **+** 号 → **New repository**
2. Repository name 填：`astock-scanner`
3. 选择 **Public**（必须是 Public，免费版 GitHub Pages 只对公开仓库开放。介意的话：页面已设置 noindex 不被搜索引擎收录，且网址没人知道）
4. 其他都不用动，点绿色按钮 **Create repository**

## 第三步：上传项目文件

1. 先把我给你的 zip 在电脑上解压
2. 回到刚建好的仓库页面，点 **uploading an existing file** 这个蓝色链接
3. 打开解压后的文件夹，**全选里面的所有文件和文件夹**（src、tests、docs、data、requirements.txt、README.md），拖进浏览器的上传区域
4. 等文件列表出现后，点底部绿色按钮 **Commit changes**

> 注意：`.github` 文件夹因为名字以点开头，在 Windows/Mac 上是隐藏的，拖拽经常会漏掉，所以下一步手动创建它。

## 第四步：创建两个自动任务文件（复制粘贴）

**文件 1：**

1. 仓库页面点 **Add file** → **Create new file**
2. 文件名框里输入（一字不差，斜杠会自动变成文件夹）：
   `.github/workflows/daily.yml`
3. 把解压文件夹里 `workflow文件内容/daily.yml.txt` 的**全部内容**复制粘贴到大输入框
4. 点右上 **Commit changes** 两次确认

**文件 2：** 同样方法，文件名输入 `.github/workflows/backtest.yml`，内容复制自 `workflow文件内容/backtest.yml.txt`。

## 第五步：首次运行（下载历史数据）

1. 点仓库顶部的 **Actions** 标签页（如果有提示按钮，点绿色的启用按钮）
2. 左侧点 **每日扫描** → 右侧点 **Run workflow** → 再点绿色 **Run workflow**
3. **首次运行要下载 5000+ 只股票一年的历史数据，需要 1~5 小时**，之后每天只需几分钟。你可以关掉网页去干别的。
4. 运行结束后图标变成绿色 ✓。如果是红色 ✗：点进去 → 点 scan → 展开报错的步骤，把红字截图发给 Claude，我来修。数据源偶尔限流，失败了**直接再 Run workflow 一次**，会从断点继续。

## 第六步：开通网站（GitHub Pages）

1. 仓库顶部 **Settings** → 左侧菜单 **Pages**
2. Source 选 **Deploy from a branch**
3. Branch 选 **main**，旁边的文件夹选 **/docs**，点 **Save**
4. 等 1~2 分钟刷新页面，顶部会显示你的网址：
   **`https://你的用户名.github.io/astock-scanner/`**
5. 手机浏览器打开这个网址，加个书签，以后随时看。

## 第七步：跑一次回测（强烈建议）

Actions → 左侧 **策略回测** → **Run workflow**。十几分钟后回测页会显示两套规则过去一年多的胜率、平均单笔收益、最大回撤，以及同期沪深300 基准。**先看回测结果，再决定要不要按信号操作。**

## 必须知道的限制（诚实声明）

1. **回测好 ≠ 未来能赚**。历史统计只说明规则过去的表现，不构成投资建议。
2. **T+1 风险**：信号日次日开盘买入，最快再下一日才能卖，隔夜跳空无法止损。回测已按此规则模拟。
3. 免费数据源（东方财富/新浪，经 AKShare）偶尔限流或改字段。程序遇到字段变化会报**明确的错误信息**而不是静默算错——截图给 Claude 即可修复。
4. 除权除息日价格会跳变，程序每周对每只股票做一次前复权校正，极端情况下个别股票当周信号可能失真。
5. 基本面里的"应收账款、商誉"两项因免费接口口径问题暂记中性分（各占 5% 权重），其余六项指标正常计算。
6. "净利润大幅下滑"信号基于**净利润同比**（东财批量接口没有扣非口径），与扣非的差异是非经常性损益。
7. 回测中：换手率用历史真实数据精确回测；**市值用当前市值近似**（无免费历史市值数据）；基本面按交易日当时已披露的报告期匹配，不用未来数据。
8. 法定节假日程序会自动识别休市并跳过，不会写入假数据。
